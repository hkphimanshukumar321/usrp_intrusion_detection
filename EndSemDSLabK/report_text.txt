1. Introduction
Software Defined Radio (SDR) has transformed the way wireless communication and
sensing systems are designed and deployed. This report presents the design and imple-
mentation of a real-time physical intrusion detection system (PIDS) using SDR hardware
(USRP B210) and GNU Radio. The system functions similarly to a passive radar, relying
on signal reflections caused by movement or intrusion in a monitored area.
2. System Overview
The proposed system operates by transmitting a known waveform using a USRP B210,
receiving the reflected signal, and analyzing it for anomalies. It functions through the
following pipeline:
Figure 1: Overall GNU Radio Flowgraph
• Continuous waveform generation and QPSK modulation.
• Transmission using USRP B210.
• Reception of the reflected signal.
• Signal processing: AGC, filtering, demodulation, and detection.
• Motion detection using peak detection and thresholding.
• Audio alarm triggering.

3. Transmitter Design
Figure 2: Transmitting Block
3.1. Signal Generation
A cosine wave of 1 kHz is generated with a sample rate of 1.92M. This baseband signal
serves as the signature waveform.
3.2. Modulation
The signal is modulated using QPSK with no differential encoding and a sample-per-
symbol (sps) value of 10. Excess bandwidth is set at 0.35.
3.3. Filtering and Transmission
An interpolating FIR filter upsamples the modulated signal (interpolation factor: 8). The
output is transmitted using USRP Sink at a center frequency of 2.4 GHz.
4. Receiver Design
4.1. Signal Acquisition and Conditioning
The USRP Source block captures the reflected signal using the same frequency and sample
rate. AGC ensures stable signal amplitude regardless of movement distance.

4.2. Baseband Conversion and Filtering
The signal is mixed with a local oscillator (Multiply block), then filtered and downsampled
using a Decimating FIR Filter (Decimation: 16, Taps: 64).
4.3. Demodulation
• Root Raised Cosine Filter: Reduces ISI.
• Symbol Sync: Recovers symbol timing using Mueller-Muller method.
• Costas Loop: Locks onto carrier phase.
• Constellation Decoder: Extracts bitstream.
4.4. Alarm Logic and Detection
• Peak Detector: Identifies motion-induced spikes.
• Threshold Block: Binarizes signal (e.g., TH high: 20, low: -20).
• Binary Slicer: Converts to binary (0 or 1).
• Tone Generator and Multiply: Mixes with intrusion signal.
• Audio Sink: Triggers real-time alarm tone.

5. Why USRP B210?
• Dual-channel SDR with MIMO capabilities.
• Operates in wide frequency range: 70 MHz to 6 GHz.
• Compatible with GNU Radio for prototyping.
• USB 3.0 support enables real-time data streaming.

6. Results
The system successfully identifies motion within the detection region. The reflected signal
perturbation causes the peak detector to activate, which in turn crosses the set threshold,
resulting in an audible alarm. Debugging via QT GUI Time Sink validates signal flow
and detection logic.

8. Conclusion
The developed SDR-based intrusion detection system offers a flexible and scalable
approach to motion detection in secured environments. Utilizing USRP hardware
with GNU Radio allows rapid prototyping and real-time performance, making this
approach viable for home, office, and industrial security applications.
